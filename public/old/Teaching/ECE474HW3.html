ECE 474 Homework 3

Verilog synchronous circuits and testbench fundamentals

Background:

We will now experiment with synthesizable Verilog constructs for creating synchronous circuits. We will build a clock divider (also known as a frequency divider), place it in a testbench and test it using some non-synthesizable verilog.

We will also use some verilog constructs for building a testbench and use them to test our counter.

We will NOT be synthesizing circuitry this time.

You will create a variable width divider. Its features are as follows: 

  -asynchronous reset
  -asynchronous enable (the divider may still be running but the output will be 0)
  -divide value N changes the output frequency synchronously to 
   the out clock. i.e. the next value of N will not change the 
   frequency until the next rising edge of out
  -can divide by only even numbers for now
  -has a default 8-bit divide by number (N)
  -it will be parameterizible

Work to do:

1) Create a hw3 working directory. 

   Create a work directory.
   wget the files tb_skel.sv, and divider.out

2) Code the divider as divider_even.sv. The module must look as follows: 

  module divider_even #(parameter WIDTH=8)(
    input clk,              // fast input clock
    input [WIDTH-1:0] N,    // divide by factor 'N'
    input reset_n,          // asynchronous reset
    input enable,           // enable the even divider
    output out              // slower divided clock
  );

Function:

  • An input clock with 50% duty cycle will be connected to the input clk.
  • input N is synchronous to the out clock.
  • The divider can divide by any even number.
    • When 0 is input to N the divider_even module outputs logic 0.
    • When the input is an odd number the divider_even module rounds N down to the next even number.

3) Code a Verilog testbench to test your divider called tb.sv. Start with the skeleton code I have given you. The clock will run at 100Mhz. Apply control signals to the divider using the verilog delay (#) format. Control signals will be applied 100ps after the rising edge of the clock. Create the timing scenario as shown in the attached timing diagram.

4) Compile divider_even.sv and tb.sv with the divider instantiated within it. Simulate tb to make sure your divider works as expected. I have supplied my output for you to check yours.

5) Build a doit.sh script to automate the compilation and simulation steps.

What to turn in:

A tar file (hw3.tar) consisting of: 

  divider_even.sv
  tb.sv
  doit.sh 
  divider.out

  This is an electronic submit.

Grading:

   Verilog coding of the divider      -45%
   Verilog coding of the testbench    -45%
   doit script                        -10%


   Make sure this works, it will be used in the next homework!
