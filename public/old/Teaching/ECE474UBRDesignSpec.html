ECE 474/574 Design Problem (Spring 2013)

Ultimate Bike Reflector (UBR)

Background:

Bike reflectors serve an important purpose. Specifically the reflector in the wheel while spinning can help cars see the bike from the side while moving.

There is just one problem: Bike reflectors are boring. In this project you will design SystemVerilog code that makes a more entertaining, electronic, reflector that illuminates a path as it spins on the wheel using the principal of persistence of vision (POV).

Operation:

UBR consists of three parts:

  1. A sensor that measures how fast the bike wheel is spinning.
  2. LED (or LEDs) that will blink a pattern stored in memory.
  3. Logic that you will design to take data from the sensor and blink the LED(s) based on a pattern stored in memory.

For every rotation of the wheel, 32 "pixels" will be displayed equally spaced across the circumference by turning the LED(s) on or off at the appropriate time. The pixel information is stored in an 8-bit read-only memory. For every rotation UBR will need to read 4 bytes from memory and display them in the correct order.

UBR Interfaces:

The interface of UBR to the outside world is as follows: 

          INPUTS                           OUTPUTS
                       _______________
                      |               |
          in_sensor --|               |-- out_LED[N:0]
                      |               |
                      |      UBR      |
   in_ROM_DATA[7:0] --|               |-- out_ROM_ADDR[3:0]
                      |               |
                      |               |-- out_ROM_RE
                clk --|>              |-- out_ROM_CE
            clk_mem --|>              |
            reset_n --|o              |
                      |_______________|

  • clk (Main system clock)

   ______/     \______/     \_____
          <----------> 
                |
              frequency: 100MHz
              duty-cycle: 50%
  • clk_mem (Memory clock) 
          _____        _____
   ______/     \______/     \_____
          <----------> 
                |
              frequency: 25MHz
              duty-cycle: 50%
  • reset_n (Asynchronous reset, active low) 
   _______________________        ________
                          \______/     
                           <----> 
                             |
                           width:
                              min: 1 cycle of clk
                              max: infinity
  • in_sensor (Sensor indicating full rotation of wheel) 
             _____                          _____
   _________/     \________________________/     \_____
             <--->  <--------------------->
               |               |
               |             width:
               |                min: 256 cycles of clk
               |                max: unknown
             width:
                min: 2 cycles of clk
                max: unknown
  • out LED (single- or multi-bit bus to drive LED(s)) 
   No special timing requirements.
