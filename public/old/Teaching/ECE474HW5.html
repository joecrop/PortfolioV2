ECE 474 Homework 5

Design Project Synthesis

Background:

We will now add synthesis constraints to the design project and rerun the simulation. You will also get more output from the synthesis tool to interpret. The RTL design from the last assignment will be reused. No HDL coding will be required in this assignment.

The Assignment:

You already have: 

  -a hopefully correct RTL implementation of the design 
  -a working doit script

You will be given: 

  -skeleton synthesis script tcl code 
  -a new do file

Use a bash script called "doit" to run everything. As before, it should.... 

  -compile your rtl image
  -run the rtl simulation and use the simulator tcl
   script to create the output vector file (golden_results).
  -synthesize UBR (sensor.sv) using your edited synthesis script
  -compile the gate-level image of UBR into ModelSim
  -run the gate level simulation and regress against the golden_results file
  -using some unix tool within your script, parse the area.txt file to look
   for the accidental inclusion of latches.  Have your doit script report
   any found.

In the process, you will create several reports. All the reports generated will be in a directory called "reports". This directory will contain: 

  reports/
    -area.txt             --number of gates and cells used
    -delay.txt            --setup time slack report (two worst paths)
    -hold.txt             --hold time slack report (two worst paths)
    -rtl_miscompares.txt  --rtl simulation differences from diff
    -gate_miscompares.txt --gate "no delay" simulation differences from diff

You will also create a synthesis log file. Put it in a directory called "logs". 

  logs/
    -synth.log            --synthesis transcript from DC

The top-level design netlist will be placed in vlogout: 

  vlogout/
    -sensor.gate.v           --the design in gate-level structural verilog

An sdf file will be created and it will be placed in sdfout: 

  sdfout/
    -sensor.gate.sdf           --timing info for vsim

Put your Verilog source code in a directory called "rtl_src".

Rename your synthesis script, "dc_syn". Put it in a "bin" directory. We will be placing other scripts inside there. The "doit" top level script stays at the top level.

Work to do:

  1. Complete the skeleton synthesis script, adding information where the lines begin with "##" (replace ## with your code)
  2. Write the doit script to automate all the steps required.
  3. Make the gate level design work. Make sure the output data compares without error.
  4. Once the gate level design works, add timing with the inclusion of the sensor.gate.sdf file.
  5. Make sure you have no setup or hold time errors by examining the output from dc_shell.

What to turn in:

A single tarred and gzipped directory of your hw5 (that includes everything) plus: 

  A README file with 
     -Total gate count and area
     -Worst case setup time in design and its associated path 
     -Worst case hold time in the design and its associated path
     -Number of latches found if any and how you got rid of them if any

Grading:

  -working gate level UBR          45%
  -TCL DC synthesis script         25%
  -doit script                     20%
  -readme file                     10%

ERATTA:

1. There is a bug in the .sdf file produced by synopsys or (most likely) in the 

   library file.  There are two hold time parameters related to reset that are 
   not found.  So, you must use sed to remove these lines in your doit script
   before you use the sdf file.  Here is the fix:

   #fix missing timing parameter in sdf file
   sed '/(HOLD (posedge RSTB)/d' < sdfout/sensor.gate.sdf  > sdfout/$$tempfile
   sed '/(HOLD (posedge SETB)/d' < sdfout/$$tempfile > sdfout/sensor.gate.sdf

2. To make all the list files match, you will need to run the simulator in 

   nanosecond mode.  So the both vism invocations must include "-t 1ns".

3. Make sure you use the new .do file.

4. Example simulation commands 

   Simulating with verilog gates and no sdf file:

   echo "***** Simulating Gate-level Verilog top *****"
   vsim top -do sensor.do  +nowarnTFMPC -quiet -c -t 1ns 
   mv vectors/sensor.list vectors/sensor.gate.list



   Simulating with verilog gates and including sdf file:

   echo "***** Simulating Gate-level Verilog top with sdf file*****"
   vsim -novopt top -do bin/sensor.do +nowarnTFMPC -sdfmax /top/sensor_0/=sdfout/sensor.gate.sdf -t 1n
   mv vectors/sensor.list vectors/sensor.gate.sdf.list
