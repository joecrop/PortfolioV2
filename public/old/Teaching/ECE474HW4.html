ECE 474 Homework 4

Design Project

Background:

In this lab we will take the system design (UBR) we designed in class and design the data paths as well as the state machines for controlling them.

The Assignment:

I will give you a a top level block diagram of the design. What you need to do is fill in the blocks with the appropriate logic. This first requires a complete logic design.

Work to do:

  1. Finish designing any other parts of the design. Then code the modules. Remember: "coding" is not the same as "designing"! "If you can't draw it, you can't code it." Debug the design at the sub-block level, if you try to piece it all together all at once it will NOT work!
  2. See that you obtain correct operation. Using your list file output and the golden_results file prove correct operation. A ubr.do file is provided to check your work.
  3. You are also provided with a top.sv and rom.sv that should be compiled with your design. You will code the ubr module and test it from within my top.sv.
  4. The compiling, running, simulation and comparison should be orchestrated by a doit script as before. There is no synthesis or testbench for this assignment, only pure RTL Verilog.
  5. Your code must run under my shell script and against my golden_results file for full credit.
  6. You will need a directory "vectors" for the list file to write your results to. Make that in your doit script.

What to turn in:

  • Submit state machine diagrams for all your state machines as scanned .pdf files in your hw4 directory.
  • Put all your verilog code in an rtl_src directory.
  • Submit your entire hw4 directory by ZIP file to the TEACH website.
  • Grading 
   -correct operation with correct answers   %80
   -correct state machine diagrams           %20
