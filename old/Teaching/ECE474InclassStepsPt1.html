login to Linux machine such as flip.engr

make a working area, for example: mkdir incalss1

wget the files needed:

wget https://web.engr.oregonstate.edu/~traylor/ece474/inclass/wk1/fadder.sv
wget https://web.engr.oregonstate.edu/~traylor/ece474/inclass/wk1/comp_lib.sh
wget https://web.engr.oregonstate.edu/~traylor/ece474/inclass/wk1/doit.sh
wget https://web.engr.oregonstate.edu/~traylor/ece474/inclass/wk1/.synopsys_dc.setup

for help on vi, see: https://web.engr.oregonstate.edu/~traylor/ece473/lectures/programming_editor_vi.pdf

make work directory for vsim: vlib work
compile sverilog file: vlog fadder.sv
simulate the rtl version of fadder: vsim -novopt fadder

To do the simulation you will need to tell the simulator (in transcript window):

view signals
add wave -r /*
force a 0
force b 0
force cin 0
run 10ns

Force a and or b to logic one and step the simulator forwards another 10ns to view the results. Play with setting the value of a an b and seeing the results.

Under the Transcript window select File > Save Transcript As...

Save the transcript with a useful name like "fadder.do"
Now quit the simulator. Let's use the do file to run the simulation.

The new "fadder.do" file is created in the same directory as fadder.sv.
vi the file and check out what's inside.
Restart simulator with do file argument:

vsim -novopt -do fadder.do fadder

Take a look at the results. Zoom in and observe the behavior. It should look exactly as it did the first time with the wave stimuli applied.

Note that the simulator has a shell interpreter much like the Linux prompt. At the simulator prompt, you can type: ls, pwd, date, etc as in the bash shell prompt.

Then quit via: quit -f

What you have done is to observe the behavior of a verilog half adder module. The source file was compiled to a form the simulator could run. We then applied stimulus to the design with a "do file" and observed the results.

The design has no gates in it at this point. There are no propagation delays either. Now, lets move to a synthesized design.

Create a synthesis (TCL) script with your editor called syn_script. In it, put the following:

read_sverilog fadder.sv
compile
report_timing
report_area

Invoke the synthesis tool: design_vision-xg -f syn_script

It will read the TCL script we just wrote and synthesize the circuit and produce the reports. Look at the synthesized design by clicking the gate symbol near the top of the window under the View pulldown.

Read the information produced about the size of the design and its delays and look at the schematic. You can see the information produced in the invoking window or within the tool gui. Look at the number and type of cells used by typing report_cell in the command window.

Add a line at the end of your syn_script that says:

write -format verilog -hierarchy -output fadder.gate.v

Change the line that says compile to compile_ultra.

Source syn_script from command line to make another synthesis run.

e.g.: source syn_script

See new reports and note any improvements.
