ECE 474 Homework 6

Design Project Synthesis

Background:

We will now take the synthesized design and place the design using a "place and route" tool.

The Assignment:

You already have: 

  -a hopefully correct RTL implementation of the design 
  -a working do file for vsim
  -a synthesized gate-level verilog file and sdf file.

You will be given: 

  -skeleton place-and-route script tcl code

Use a bash script called "doit" to run everything. As before, it should.... 

  -compile your rtl image
  -run the rtl simulation and use the simulator tcl
   script to create the output vector file (golden_results).
  -synthesize UBR (sensor.sv) using your synthesis script from hw5
  -compile the gate-level image of UBR into ModelSim
  -run the gate level simulation and test agains the RTL simulation
  -take the gate-level verilog and sdf file from Design Vision
   and place the design using SOC Encounter
  -simulate the placed and routed design in modelsim and
   test against the RTL simulation

In the process, you will create several reports. All the reports generated will be in a directory called "reports". This directory will contain (along with the synthesis reports): reports/ 

    -power_pnr.txt             --total static power dissipation
    -setup_pnr.txt             --setup time violations report
    -hold_pnr.txt              --hold time violations report
    -conn_pnr.txt              --connectivity violations report
    -geom_pnr.txt              --geometry violations report
    -antenna_pnr.txt           --antenna violations report

You will also create a place/route log file. Put it in a directory called "logs". 

  logs/
    -pnr.log            --pnr transcript from Encounter

The top-level design netlist will be placed in vlogout: 

  vlogout/
    -final.v           --the design in gate-level structural
                         verilog from encounter

Don't forget the sdf file! 

  sdfout/
    -final.sdf

You will need to change primarily "pnr/encounter.tcl" to place and route the design.

Work to do:

  1. Complete the skeleton encounter.tcl script, adding information where the lines begin with "##" (replace ## with your code)
  2. Write the doit script to automate all the steps required.
  3. Make the gate level PNR design work. Make sure the output data compares without error.
  4. Once the gate level PNR design works, add timing with the inclusion of the final.sdf file.
  5. Make sure you have no design violations or errors by examining the output from Encounter.

What to turn in:

A single tarred and gzipped directory of your hw5 (that includes everything) plus: 

  A README file with 
     -Total density
     -Worst case setup time in design and its associated path 
     -Worst case hold time in the design and its associated path
     -Any physical violations in your design

Grading:

  -working PNR gate level UBR      45%
  -encounter TCL script            25%
  -doit script                     20%
  -readme file                     10%

ERATTA:

1. There is a bug in the .sdf file produced by encounter or (most likely) in the 

   library file.  There are two hold time parameters related to reset that are 
   not found.  So, you must use sed to remove these lines in your doit script
   before you use the sdf file.  Here is the fix:

   #fix bad timing parameter in sdf file
   sed '/(WIDTH (negedge RSTB)/d' < sdfout/final.sdf  > sdfout/$$tempfile
   sed '/(WIDTH (posedge CLK)/d' < sdfout/$$tempfile > sdfout/$$tempfile2
   sed '/(WIDTH (negedge CLK)/d' < sdfout/$$tempfile2 > sdfout/final.sdf

2. To make all the list files match, you will need to run the simulator in 

   nanosecond mode.  So the both vism invocations must include "-t 1ns".

4. Example PNR commands 

   Running place and route:

   echo "***** Running place and rout *****"
   cd pnr
   ./dopnr.sh
   cd ..
